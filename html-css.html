CSS Positioning

The 5 position properties that elements in CSS can have are:

  • Static - the default position of any element. Which just appears in the order that it appears in the code.
  • Relative - will adjust its position on screen relative to where it would normally appear, once given the parameters to do so.
  • Absolute - will position itself in a specified position within the nearest parent element which also has an absolute or relative positioning.
  • Fixed - will fix an element to position in the viewport (screen) and will stay in the same position even as you scroll.
  • Sticky - similar to fixed, except this element can appear as though it were static (normal) until you scroll past it, which is when it gets fixed to a specified position on screen.

Let's focus on the Relative, Absolute & Fixed positions.

Relative

An element with this property may move relative to its normal default position. If set to relative without any additional parameters then it will just appear in its default position:

No parameters

You can add the parameters top, right, bottom, left, which will move the element the specified amount away from that side i.e. "top: 30px" will move the box 30px down from where it would have appeared by default. "left: 25%" will move it 25% of it's parent element (the white container) away from the left side of where it would have appeared by default:

top: 30px

left: 25%

Absolute

An absolute element allows you to position it absolutely anywhere you want to on the page.

It will sit at a specified position within it's nearest parent element that has "position: relative" or "position: absolute". If no such parent element exists, then it will use the whole document as it's nearest parent/point of reference.

Strangely without parameters it will still sit within its parent element whatever the parent elements position.

See below, the blue box is absolutely positioned inside of the red box which has "position: relative". It's sitting in the top left corner because it has no extra parameters.

Use the dev tools (ctrl+shift+i) to see what happens when you turn off the relative position of the red-box

And here, the "property left: 25%;" moves the blue square away from the left by 25% of it's parents element (the red square). While the property top: 50%; shifts it away from the top by 50% of the parent element.

Inspect the blue-box with Dev tools & play with the top & left properties. What happens when you use a negative number?

I've found this concept very confusing in the past. While you might think that giving an absolutely positioned element the properties "left: 50%; top: 50%;" would center the blue box inside the red box, it does not.

It shifts the left side of the box to the horizontal 50% point of the red box, and the top side of the blue box to the 50% point of the red box.

While in that example you could use "position: 25%;" to center it since the blue box is exactly half the size of the red box, that would actually make things very confusing later when you try to use absolute positioning to center an element which has proportions at odds to its parent.

So to solve this, you can center the blue box by adding the property "transform: translate(-50%,-50%);" which shifts the blue box along the X & Y axis by "-50% of its own size".This took me a long time to get my head around, and it wasn’t until writing this blog post that it clicked.

Play around with the translate properties in your DevTools. Turn it off and on, try out different percentages, pixels and substitute 0 in for one of the values.

Absolute is a powerful property, but it’s important to remember that when an element has "position: absolute;" it's removesd from the flow of the document, therefore it cannot affect other elements, and other elements cannot affect it.

Fixed

If an element is fixed then it will be fixed in a specified position on your screen, even as you scroll.

I’ve illustrated this with the thin green line which is hovering 2px from the bottom of your screen.

Fixed accepts the same parameters as relative & absoulute (top, bottom, left, right). But in this case they position the element a specified amount away from the edges of the viewport (screen). This is handy if you want the navigation to stay at the top of the screen as users scroll, if you want to have a chat bubble in a corner of the screen, or something like an accessibility toolbar on the side of the screen at all times.

The biggest thing to keep in mind when using "position: fixed;" is hat it can interfere with content on your page generally by obscuring it, but sometimes content can also appear over top of your fixed element. So do make sure to test in multiple views( desktop, mobile, tablet) to ensure there isn't any interference.
