JavaScript & the DOM

Javascripts relationship to HTML & CSS

There are 3 languages of the web. HTML creates the structure and basic content of a web page, CSS adds the style, and JavaScript brings it to life.

If you were to think of a web page as a car:

HTML would be the chassis, the panels, the seats, the ignition, the buttons, the levers & the pedals.

CSS would be all of the style on your car: the cushions, the fabric, the paint, the trim, the fonts & the shapes.

JavaScript would be the engine & doer of all the things that your car can do once you turn it on. It opens the wing mirrors, makes the beeps, starts the music, accelerates when you push the accelerator, brakes when you hit the brakes, & blinks the indicators when you flick the lever.

Control flow & loops

When a piece of JavaScript code is executed there is a control flow which is how the code is controlled to flow. While it does run from start to finish when first executed, it’s very common that there will be some pieces of code which redirect the control flow to do something else before carrying on with running the rest of the code in order.

For example if I was running a program to mow the lawns, when I get to the lawnmower I check the petrol, and if there isn’t enough petrol in there, it will take me out of the ordinary flow of mowing the lawns because I would then have to execute another “function” before the lawns can be done.

Filling up the lawn mower, it seems to happen linearly in time because I am flexible, but because a script of code is a linear document, the function for filling up would be stored in another part of the document, which ensures that the lawn mowing function is tidy, so the control flow would go run that function, before coming back to finish taking the lawnmower out of the garage.

To change the flow the script uses many different control structures like if…else statements, conditionals, loops, & functions.

The previous example was an if…else statement, if (there’s not enough petrol) {Fill it up}. Which would then run a loop function to fill up the petrol, starting at empty, I would continue to run the function of pouring petrol into the tank, until the condition of full is met, at which point I exit the loop and carry on with the rest of the mowing lawns function.

There may even be lines of code waiting for events to happen to run another set of code, for example, while mowing the lawns, my wife might come out and say “Hey can you give me a hand to lift this table?”, which is like pushing a button on my DOM that pauses mowing the lawns & starts a completely different helpfulHusband function.

What is the DOM & how might you interact with it?

“DOM” stands for Document Object Model.

Whenever you go to a webpage, the browser creates a DOM of that page.

It helps if you understand what an object is in JavaScript. As that is essentially what the Document Object is. But this object is huge, and includes everything that your webpage is made up of.

The DOM is what enables you to interact with the web page. When you push something, scroll, swipe, enter data or have any other interactions with the page, the DOM is what facilitates that. It allows programs & scripts to access & update the structure, style, content within a web page.

You can inspect the DOM using DevTools (ctrl+shift+i).

To see the document object in all its glory, within the DevTools console you can type “console.log(document.all)”.

Or after you type “console.log(document.” you should see a list of all the things within the document object that you can inspect.

Then you can interact with the DOM from your JavaScript file in much the same way, by referencing “document.” and the part of the document you would like to access.

Some of the most basic but powerful ways to use the DOM from JS are:

document.getElementsByClassName(“myClassName”)

Which will return an array containing all the elements in the document with the class “myClassName”.

document.queryselector(“#getMeByMyId”).onclick = myFunction

Which as that indicates by the “#” would point to the HTML element in the document with the id “getMeByMyId”. This “.onclick” method would tell the document to “listen” for a click on the element with this id, & when clicked would run “myFunction”.

Event.target

This is one of my favourites. After listening for a click, you can then access that event “the click” with “event.target”. To see what this returns at what it gives you access to within the DOM, you can add “console.log(”event.target”)” inside the function that gets called (i.e. “myFunction” from the example above), as long as when declaring the function you also put “event” within the () i.e. `myFunction(event) {

console.log(”event.target”)}. This is particularly helpful when you need information from the element being clicked to know which data to access with your JS.

Accessing data from arrays & objects

Arrays & objects are a great way to store data, so how do you access that data when you need it?

Arrays

The simplest way to access data from a specific position in an array is with:

Array[i]

Where i is the index of the array. I.e array[2] accesses pig from:

myArray = [dog, rabbit, pig, dinosaur]

Notice that in arrays you count from 0.

Dog 0

Rabbit 1

Pig 2

Dinosaur 3

There are also some array methods that you can use to manipulate the data in more complex ways.

Objects

The simple way to access data inside objects is to use the property name to reference the position the data is stored.

You can do this in two ways:

objectName.propertyName

Or

objectName[“propertyName”]

The first way being the simpler & tidier option. But the second way can be extremely useful. The reason that “propertyName” is contained within quotation marks here, is because you could also reference a variable inside of the [], which is incredibly handy as you can store received data from the DOM as a variable to then access an object property with that variable.

You can build a function to do something over and over again, whenever you ask it too.
